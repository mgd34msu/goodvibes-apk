# GoodVibes Companion v0.3.0

SDK compatibility release for GoodVibes SDK 0.26.10.

## Highlights

- Bumps `@pellux/goodvibes-sdk` to **0.26.10**
- Keeps normal mobile session replies explicit with `kind: "message"` for the SDK 0.26 message-routing behavior
- Handles the SDK's new conversation-route response shape for shared-session messages without treating it like a queued agent submission
- Moves companion chat session create/get/update/message calls onto the public operator contract invocation surface
- Keeps provider runtime event typing derived from the public SDK realtime union instead of the full-surface `platform/*` event path

## Behavior Notes

- Shared/TUI session model changes still use the daemon's global current-model endpoint by design
- Companion-only remote chat sessions still keep provider/model selection scoped to that chat
- This app remains a remote client; it does not run the GoodVibes runtime locally on the phone

## Setup

You will need:

- a running GoodVibes daemon compatible with SDK 0.26.10
- a reachable daemon URL such as `http://192.168.x.x:3210`
- daemon username/password credentials or a bearer token with the scopes you need

## Known Constraints

- No iOS build artifact in this release
- Phone must be able to reach the daemon URL directly
- Approval, session-write, and chat behavior still depend on the scopes granted by the daemon auth in use
