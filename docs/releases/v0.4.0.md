# GoodVibes Companion v0.4.0

SDK compatibility release for GoodVibes SDK 0.27.0.

## Highlights

- Bumps `@pellux/goodvibes-sdk` to **0.27.0**
- Keeps the APK on the public React Native/client SDK surface
- Preserves the SDK 0.26 companion chat routing behavior added in v0.3.0
- Keeps release builds aligned with the current GitHub Actions APK artifact flow

## Behavior Notes

- This app remains a remote companion client; it does not run the GoodVibes runtime locally on the phone
- Shared/TUI session model changes still use the daemon's global current-model endpoint by design
- Companion-only remote chat sessions keep provider/model selection scoped to that chat

## Setup

You will need:

- a running GoodVibes daemon compatible with SDK 0.27.0
- a reachable daemon URL such as `http://192.168.x.x:3210`
- daemon username/password credentials or a bearer token with the scopes you need

## Known Constraints

- No iOS build artifact in this release
- Phone must be able to reach the daemon URL directly
- Approval, session-write, and chat behavior still depend on the scopes granted by the daemon auth in use
