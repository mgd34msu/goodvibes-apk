# GoodVibes Companion v0.1.0

First public Android release of the GoodVibes mobile companion.

## Highlights

- Secure mobile companion for the GoodVibes daemon with password or bearer-token sign-in
- Android QR onboarding for fast daemon setup on a phone
- Remote chat sessions backed by the daemon, including persisted companion-only conversations
- Shared-session browsing with transcript viewing, replies, and follow-up messages
- Approval review and action flow for pending daemon approvals
- Provider and model visibility, including provider warnings, model changes, and current-model switching
- Realtime foreground updates plus a mobile activity feed for important daemon events

## What You Can Do

- Connect the app to a reachable GoodVibes daemon over LAN or another routable network
- Review control-plane state, task visibility, open shared sessions, and approval backlog
- Open active shared sessions, inspect their transcript, and send new input without returning to the terminal
- Create or resume companion-only remote chat sessions on the daemon
- Inspect configured providers and switch the current model when the daemon allows it

## Android Notes

- The release includes a signed installable APK attached to this GitHub release
- QR scanning is available on Android builds
- The app is a remote client; it does not run the GoodVibes runtime locally on the phone

## Setup

You will need:

- a running GoodVibes daemon
- a reachable daemon URL such as `http://192.168.x.x:3210`
- either daemon username/password credentials or a bearer token with the scopes you want to use

## Known Constraints

- This release does not include an iOS build artifact
- The phone must be able to reach the daemon URL directly
- Approval actions, session writes, and chat behavior depend on the scopes granted by the daemon auth you use
